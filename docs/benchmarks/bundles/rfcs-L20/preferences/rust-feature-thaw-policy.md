# 규칙

- 기능이 stable Rust에 처음 실려도 바로 쓰지 않는다. 그 릴리스로부터 릴리스 4번이 더 지나야
  "해동됐다"고 보고, 그때부터 프로덕션 코드에 사용한다.
- `Cargo.toml`의 `rust-version` 필드는 이 규칙 때문에 항상 현재 stable보다 4 낮게 잡힌다.
- MSRV(`rust-version`) 숫자를 올리는 작업은 릴리스마다 따라가지 않고, **분기 첫 스프린트에서만**
  진행한다.

# 이 숫자의 근거 (반드시 함께 기억할 것)

4라는 대기 릴리스 수와 분기 1회라는 갱신 주기는 **공급망 보안이나 크레이트 쿨다운과 무관하다**.
순전히 **사내 빌드 이미지에 박힌 rustc를 분기에 한 번만 업그레이드하기 때문에** 생기는 숫자다.
이 인과관계를 빼고 "4번 지나야 해동"이라는 결과만 남기면, 다음 세션이 업계 관행(예: 크레이트
쿨다운 정책)으로 오인하고 잘못 방어하게 된다.

# 예외 두 가지

1. **`xtask/` 크레이트**: 배포 산출물에 포함되지 않으므로 해동 규칙 면제 — stable 되는 날 바로
   사용해도 된다.
2. **unsafe 제거가 가능해지는 기능**: 대기하지 않고 stable 되자마자 즉시 도입한다. 위험한
   unsafe 코드를 오래 들고 있는 비용이 새 기능을 조기 도입하는 비용보다 크다고 판단했기 때문이다.

# 2026-07-16 기준 조사된 후보 RFC의 해동 상태

사내 rust-lang/rfcs 미러의 cargo/MSRV 관련 RFC를 이 규칙에 대입해 조사한 결과:

| RFC | 내용 | 상태 |
|---|---|---|
| 2495 | `Cargo.toml`의 `package.rust-version` 필드 | 1.56.0(2021-10)에 stable, 이미 해동 완료 — 지금 프로덕션에서 사용 가능 |
| 3537 | MSRV 기반(`v3`) 리졸버 | 아직 unstable, 진행 중 |
| 3923 | `min-publish-age` (패키지 공개 시간 제한) | 2026-02 시작, 아직 미확정 |
| 3692 | `resolver.feature-unification` | 일부 unstable, 진행 중 |

RFC 2495만 현재 시점에 해동되어 실제로 쓸 수 있는 상태다.
