# OKF 인덱스 개요 스킬

`<OKF_HOME>`(보통 `~/.claude/okf`) 번들에 현재 어떤 지식이 쌓여있는지 사람이 훑어보기 좋은
형태로 요약해 보고한다. `SessionStart` 게이트가 주입하는 루트 카테고리 요약보다 한 단계 더
자세한, 개별 concept 제목 수준의 개요다.

## 절차

1. `<OKF_HOME>/index.md`(루트)를 Read해서 어떤 카테고리(디렉토리)가 있고 각각 몇 개인지
   파악한다.
2. 항목 수가 0이 아닌 각 카테고리의 `index.md`(예: `/decisions/index.md`)를 Read한다 —
   여기엔 이미 각 concept의 제목과 한 줄 설명이 순수 목록으로 들어있다(frontmatter 없음).
   개수가 많아 전부 훑기 부담스러우면 최근 것 위주로 요약하되, 생략했다는 사실을 밝혀라.
3. `log.md`도 Read해서 최근 며칠간 어떤 변경이 있었는지 함께 파악한다.
4. 개별 concept 파일 본문까지 전부 Read할 필요는 없다 — 이 스킬은 "인덱스 수준" 개요이지
   전수 조사가 아니다. 사용자가 특정 항목을 더 자세히 알고 싶다고 하면 그때 해당 파일을 Read한다.

## 출력 형식

카테고리별로 묶어서, 사람이 훑어보기 좋게 bullet로 보고하라. 예:

```
OKF 번들 개요 (<OKF_HOME>)

decisions (3개)
- API 배포 정책으로 canary 방식 채택
- ...

troubleshooting (0개)
(비어있음)

최근 변경 (log.md)
- 2026-07-15: ...
```

번들 자체가 아직 비어있으면(모든 카테고리 0개) 그 사실을 그대로 보고하고, 배치가 아직 한 번도
안 돌았을 가능성(`/okf:okf-status`로 확인 가능)을 안내하라. 이 스킬은 조회 전용이다 — 번들
파일을 Write/Edit하지 않는다(그건 `okf-usage` 스킬의 쓰기 규칙을 따른다).
